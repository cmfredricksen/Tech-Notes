Simple Git Commands to Git Started

Basically I am just noting the precise steps I need to take in order to set up a new project using visual studio code, the terminal, git and github. Just trying to get things laid out in an organized way has been tricky for me. I find it important to have a solid base and to build from there...so this page is to be a part of that foundation.

Step by Step Project Setup

  1. Create a folder on your desktop
  2. Open in Visual Studio Code
  3. Add files/folders
  4. OPen Terminal
  5. git init
  6. ls -a
  7. git status
  8. add files to staging area using git add
  9. git commit -file name- -m"message on commit"
  10. git push...if it is your first time see steps below

First Time Pushing to Github

If you have not pushed any commits to github from this project before you will need to follow the steps below your first time. Once you have made your first push then you will only need to use the 'git push' command each time you want to save commits to your online repository.

  1. Go to github.com and sign in to your account.
  2. Make a new blank repository, do not add the readme file at this time.
  3. Return to the terminal and use the command $git remote add origin "link to repo" The link to the repo is listed on the repository page, you may simply copy it and paste it into the command line.
  4. git push -u origin master

Once you have connected your local project to your github repository then you can simply commit, then use git push.

Once you are set up and have your files connected to git hub, then the process of saving versions of your work is simple and streamlined. Here it is...

  1. git add
  2. git commit -m""
  3. git push

How to Clone a Repository from Github to your Local computer

  • On Github navigate to the repo you want to clone to your computer
  • Click Clone and copy the url posted there
  • Open Git Bash
  • Change Directory (cd) to the one you want to place the repo in
  • Git Init
  • Git Clone --url of remote repo--
  • Git Remote add origin --url of remote repo--

    • Once you have cloned the repo then do the git pull command to bring the files from your remote repo to the desktop.

  • Git Pull --url of remote repo--

And thats it you are all up to date and keeping each version you make along the way.

Tips

These are random things I have figured out and dont want to forget.

  • Once you have set up your initial project file, make a branch to work on. The original file will be automatically saved as 'master'.
  • Work out your issues on your new branch, commit changes and then merge with master.
  • Push changes to github.
